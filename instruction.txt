--------------------------------------------------------------------------------
2. AWS LIGHTSAIL FIREWALL — OPEN PORT 8080
--------------------------------------------------------------------------------

Lightsail has its own firewall separate from Linux ufw — open in BOTH.

Lightsail Console:
  Instance → Networking tab → IPv4 Firewall → Add rule
  Application: Custom | Protocol: TCP | Port: 8080 → Save

Linux ufw:
  sudo ufw allow 8080/tcp
  sudo ufw reload
  sudo ufw status

--------------------------------------------------------------------------------
4. APACHE VIRTUALHOST — PROXY WEBSOCKET TO REVERB
--------------------------------------------------------------------------------

Edit: /etc/apache2/sites-available/mgvcl.conf

----
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /var/www/html/mgvcl-feeder/public

    RewriteEngine On
    RewriteCond %{HTTPS} off
    RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [R=301,L]
</VirtualHost>

<VirtualHost *:443>
    ServerName yourdomain.com
    DocumentRoot /var/www/html/mgvcl-feeder/public

    SSLEngine on
    SSLCertificateFile    /etc/letsencrypt/live/yourdomain.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/yourdomain.com/privkey.pem

    # WebSocket proxy — must come BEFORE general ProxyPass
    RewriteEngine On
    RewriteCond %{HTTP:Upgrade} websocket [NC]
    RewriteCond %{HTTP:Connection} upgrade [NC]
    RewriteRule ^/app/(.*) ws://127.0.0.1:8080/app/$1 [P,L]

    ProxyPass        /app ws://127.0.0.1:8080/app
    ProxyPassReverse /app ws://127.0.0.1:8080/app

    <Directory /var/www/html/mgvcl-feeder/public>
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog  ${APACHE_LOG_DIR}/mgvcl-error.log
    CustomLog ${APACHE_LOG_DIR}/mgvcl-access.log combined
</VirtualHost>
----

  sudo a2ensite mgvcl.conf
  sudo apache2ctl configtest      # must say Syntax OK
  sudo systemctl reload apache2


--------------------------------------------------------------------------------
5. RUN REVERB AS SERVICE VIA SUPERVISOR
--------------------------------------------------------------------------------

Install supervisor:
  sudo apt install supervisor -y

Create config:
  sudo nano /etc/supervisor/conf.d/reverb-mgvcl.conf

----
[program:reverb-mgvcl]
process_name=%(program_name)s
command=php /var/www/html/mgvcl-feeder/artisan reverb:start --host=0.0.0.0 --port=8080 --no-interaction
autostart=true
autorestart=true
user=www-data
redirect_stderr=true
stdout_logfile=/var/www/html/mgvcl-feeder/storage/logs/reverb.log
stdout_logfile_maxbytes=5MB
stopwaitsecs=10
----

  sudo supervisorctl reread
  sudo supervisorctl update
  sudo supervisorctl start reverb-mgvcl
  sudo supervisorctl status

Service management:
  sudo supervisorctl start   reverb-mgvcl
  sudo supervisorctl stop    reverb-mgvcl
  sudo supervisorctl restart reverb-mgvcl
  sudo supervisorctl status  reverb-mgvcl

Live log:
  tail -f /var/www/html/mgvcl-feeder/storage/logs/reverb.log
